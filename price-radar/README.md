# 📊 Price Radar — Микросервисный Сборщик и Прогнозист Цен

Проект состоит из двух независимых сервисов, работающих локально:
1. **Backend (FastAPI, порт 8000)** — асинхронно собирает цены (BTC, ETH, SOL) с CoinGecko в базу данных SQLite (`radar.db`) и отдаёт фронтенд-дашборд.
2. **ML Service (FastAPI, порт 8001)** — принимает историю цен от бэкенда и возвращает предикты.

---

## 🚀 Быстрый запуск

### 1. Запуск Бэкенда (Порт 8000)
```bash
cd backend
source venv/bin/activate
pip install -r requirements.txt
python3 -m uvicorn app.main:app --reload --port 8000
```
Панель мониторинга будет доступна по адресу: `http://127.0.0`

### 2. Запуск ML-сервиса (Порт 8001)
```bash
cd ml
source ../backend/venv/bin/activate
pip install -r requirements.txt
python3 -m uvicorn app.main:app --reload --port 8001
```

---

## 🧔 Инструкция для ML-инженера (Напарника)

Твоя рабочая область находится в папке `ml/`. 
Контракт данных (API) уже настроен и протестирован бэкендом.

### Твоя задача:
1. Твой сервер должен слушать **порт 8001**.
2. Бэкенд делает к тебе `POST`-запрос на эндпоинт `/predict`.
3. **Входящий формат (Request):** Бэкенд присылает тебе массив последних цен (тип `List[float]`) в формате:
   ```json
   { "history": [63668.0, 63660.0, 63710.0] }
   ```
4. **Исходящий формат (Response):** Твой эндпоинт должен строго возвращать JSON с тремя полями:
   ```json
   {
     "current_price": 63710.0,
     "predicted_price": 64665.65,
     "recommendation": "BUY"
   }
   ```
   *Допустимые значения для `recommendation`: `"BUY"`, `"SELL"`, `"HOLD"`.*

### Куда писать код модели:
* Сейчас в `ml/app/main.py` стоит математическая заглушка `last_price * 1.015`.
* Создай файл `ml/app/model.py`, обучи там свою модель (LightGBM, Prophet или LSTM) на исторических данных.
* Импортируй функцию инференса в `main.py` и замени заглушку на вызов реальной модели. Тебе **не нужно** переписывать бэкенд — если контракт данных соблюден, графики и плашки на фронтенде обновятся автоматически!
